# Curriculum rules

## Depth rules

- Do not build modules around buzzwords
- Do not create placeholder lesson names that teach nothing
- Do not treat one example as enough practical coverage

## Sequencing rules

- Start with framing and mechanism before execution detail
- Move from context to price to probability to action when possible
- Put revision and invalidation near the end of the course, not as an afterthought

## Minimum planning standard

Every lesson plan should imply:

- a core idea
- a mechanism
- an applied example
- a failure mode
- a checklist or exercise

## Word count

Set a minimum target that forces seriousness without encouraging filler.
