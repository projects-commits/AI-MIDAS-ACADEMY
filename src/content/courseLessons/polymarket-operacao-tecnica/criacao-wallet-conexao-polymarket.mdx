### Aula 5: Criação de Wallet e Conexão à Polymarket

Antes de operar na Polymarket, precisas de configurar o ambiente: uma wallet na rede Polygon com fundos em USDC.e e POL para gas. Este passo é crítico — erros aqui impedem-te de operar ou, pior, expõem-te a riscos de segurança.

**Mecanismo:** A Polymarket não detém os teus fundos. Quando conectas a wallet, estás a autorizar os smart contracts a movimentar USDC.e da tua wallet. Tu manténs o controlo total — podes revogar a autorização a qualquer momento.

<BlackboardPanel title="Checklist de Setup">

**Para operar na Polymarket precisas de:**

1. Wallet (recomendado: MetaMask ou Rabby)
2. Rede Polygon adicionada
3. MATIC/POL para gas fees
4. USDC.e para trading
5. Conexão à Polymarket.com

**Setup passo a passo:**

1. Instala a extensão MetaMask no teu browser
2. Cria uma nova wallet (guarda a seed phrase offline!)
3. Adiciona a rede Polygon:
   - RPC: https://polygon-rpc.com
   - Chain ID: 137
   - Symbol: POL
   - Explorer: https://polygonscan.com
4. Compra USDC numa exchange e faz bridge para Polygon
5. Vai a polymarket.com e clica "Connect Wallet"

```
// Usando ethers.js
const { ethers } = require("ethers");
const provider = new ethers.JsonRpcProvider("https://polygon-rpc.com");
const wallet = new ethers.Wallet(process.env.PRIVATE_KEY, provider);
const balance = await provider.getBalance(wallet.address);
console.log("Saldo POL:", ethers.formatEther(balance));
```

## Aprofundamento operacional

A wallet deve ser tratada como uma conta operacional, nao como um acessorio do browser. Ela concentra capital, permissao de assinatura, autorizacoes e historico de execucao. Quanto mais automatizado fica o processo, mais importante se torna separar carteiras por finalidade: estudo, testes, operacao manual e automacao.

O operador que usa a mesma wallet para tudo aumenta a superficie de erro. Uma permissao antiga, uma extensao comprometida ou um script mal configurado pode afetar mais capital do que deveria. A resposta nao e paranoia; e compartimentalizacao.

### Regras praticas

- Use uma wallet dedicada para a academia e para testes.
- Nunca coloque seed phrase, private key ou API secret em nota, print ou repositorio.
- Revise autorizacoes periodicamente.
- Comece com valores pequenos ate provar que deposito, conexao, compra, venda e retirada funcionam.

### Exercicio pratico

Crie uma ficha da sua wallet operacional com estes campos: finalidade, rede, saldo maximo permitido, ultimo teste de retirada, autorizacoes relevantes e plano de emergencia se o dispositivo for perdido.

## Auditoria aplicada

Esta aula deve terminar com uma pequena auditoria escrita. O objetivo nao e produzir uma nota bonita, mas transformar "Criação de Wallet e Conexão à Polymarket" em uma decisao verificavel. Use os conceitos centrais da aula (os conceitos principais da aula) para construir um registro que possa ser revisado depois, quando o mercado ja tiver se movido ou o evento ja tiver sido resolvido.

Comece pela pergunta operacional: o que eu consigo observar agora que muda minha decisao? Depois separe evidencia, inferencia e decisao. Evidencia e dado verificavel: preco, spread, profundidade, regra de resolucao, horario, volume, estado do book, endpoint consultado ou custo calculado. Inferencia e a leitura que voce faz a partir desses dados. Decisao e o que voce faz com a leitura: estudar mais, descartar, entrar com ordem limit, esperar liquidez, reduzir stake, usar API apenas para leitura ou classificar como No Bet.

### Protocolo de revisao

1. Escreva a tese em uma frase curta.
2. Liste os tres dados que sustentam a tese.
3. Liste o principal dado que poderia invalidar a tese.
4. Defina o custo de estar errado: perda financeira, slippage, fee, tempo preso, erro de resolucao ou risco operacional.
5. Defina o gatilho de pausa. Se esse gatilho acontecer, a decisao deve ser reavaliada antes de qualquer nova ordem.

### Perguntas de controle

- O mercado esta suficientemente liquido para o tamanho que eu quero usar?
- O preco atual ainda paga o risco depois de spread, fees e slippage?
- A regra de resolucao esta clara o suficiente para eu aceitar exposicao?
- Se eu estivesse explicando este trade para outra pessoa, conseguiria separar dado de opiniao?
- Existe um caminho simples de saida se a leitura estiver errada?

### Saida esperada

No fim da aula, salve uma nota de cinco linhas: mercado observado, preco observado, risco principal, decisao tomada e motivo para revisar depois. Esse habito cria memoria operacional. Sem memoria, o operador repete erros com mais confianca. Com memoria, cada aula vira uma parte do sistema de pesquisa.
