# Lesson Writer

Use this skill to draft lessons that feel like research-grade teaching, not thin content marketing.

Read `references/lesson-template.md` first. Read `references/writing-style.md` before drafting or rewriting prose.

## Workflow

1. Start from the lesson title, course context, and learning objective.
2. Identify the mental model behind the lesson.
3. Explain what the concept is and why it matters.
4. Describe the mechanism clearly.
5. Add an applied football or market example.
6. Add a blackboard-style breakdown that makes the reasoning inspectable.
7. Add at least one failure mode.
8. Add a checklist.
9. Add a practice exercise.
10. End with a key takeaway.

## Required lesson parts

Every lesson must include:

- title
- core idea
- mental model
- what it is
- why it matters
- mechanism
- applied football or market example
- blackboard breakdown
- failure mode
- checklist
- practice exercise
- key takeaway

## Writing rules

- Write deep but readable
- Avoid vague statements
- Avoid tips or signal language
- Use Portuguese with English technical terms when useful
- Keep the tone serious and practical

## Expected outputs

- Full lesson draft
- Improved lesson depth
- Stronger examples
- More explicit failure modes
- More useful checklist and exercise sections

## Failure cases

- The lesson sounds smart but does not explain mechanism
- The example is generic and does not touch football or market reality
- The lesson has no failure mode or no practice exercise
- The lesson becomes motivational instead of instructional

## When not to use this skill

- Do not use it to define a course from scratch; use `course-architect` first
- Do not use it for final file packaging
- Do not use it as the final QA layer
