# Visual Learning Designer

Use this skill to turn dense teaching into inspectable visual learning blocks.

Read `references/blackboard-patterns.md` first. Read `references/component-patterns.md` when choosing which interface pattern should carry the lesson.

## Workflow

1. Find the part of the lesson or page that is conceptually dense.
2. Decide whether it needs a blackboard panel, formula block, market flow diagram, decision tree, comparison card, failure visual, or lab idea.
3. Reduce the idea to the fewest elements that still preserve reasoning.
4. Align the block with AI MIDAS visual rules: dark research lab, blackboard learning, strong contrast, orange for risk/action, gold for insight/value.
5. Make sure the visual teaches something, not just decorates the page.

## Must be able to create

- blackboard panel ideas
- formula blocks
- market flow diagrams
- decision trees
- comparison cards
- failure mode visuals
- lab ideas

## Expected outputs

- Visual block proposals
- Component-level content structure
- Diagram specs in words
- Formula and comparison layouts
- Suggestions for where a lesson should stop being pure prose

## Failure cases

- The proposal is decorative but not instructional
- The block repeats the paragraph without clarifying it
- The visual overloads the screen with too many ideas at once
- The proposal breaks the AI MIDAS design tone

## When not to use this skill

- Do not use it as the primary skill for full lesson drafting
- Do not use it for packaging final files
- Do not use it for brand-only review without a visual learning need

## Test prompts

2. Propose visual learning blocks for a lesson about invalidation and execution discipline.
