# COURSE GENERATION PIPELINE

## Purpose

This pipeline defines how AI MIDAS ACADEMY courses move from idea to publishable structured content.

## Principle

The academy does not publish raw generated text. Content must move through architecture, writing, packaging, and review.

## Pipeline stages

1. Course thesis definition
2. Category assignment
3. Persona fit check
4. Course architecture
5. Module mapping
6. Lesson specification
7. Draft writing
8. Visual learning opportunities review
9. Packaging and metadata
10. QA audit
11. Publication readiness check

## Stage detail

### 1. Course thesis definition

Define what the course teaches, what transformation it creates, and what kind of learner it serves.

### 2. Category assignment

Map the course to one primary V1 category and optional secondary relationships.

### 3. Persona fit check

Confirm which persona or personas the course serves best and how the complexity level should adapt.

### 4. Course architecture

Design the high-level progression:

- course promise
- modules
- sequencing logic
- scope boundaries

### 5. Module mapping

Break the course into modules with clear purposes and outcomes.

### 6. Lesson specification

Define each lesson before drafting. A lesson spec should include:

- title
- thesis
- objective
- core concepts
- practical angle
- likely failure modes
- exercise direction

### 7. Draft writing

Write in Portuguese with a serious, analytical, direct, practical voice. Use English technical terms only when useful.

### 8. Visual learning opportunities review

Identify where diagrams, tables, blackboard summaries, frameworks, or annotated breakdowns can improve comprehension.

### 9. Packaging and metadata

Package the course into the project content system with consistent slugs, frontmatter, taxonomy, and output rules.

### 10. QA audit

Review for:

- anti-hype compliance
- conceptual clarity
- structural completeness
- practical usefulness
- voice consistency
- visual clarity opportunities

### 11. Publication readiness check

Confirm that the course is coherent, navigable, and aligned with AI MIDAS identity before making it public.

## Hard rules

- Do not skip architecture and go directly to prose
- Do not publish raw AI output
- Do not use promotional betting language
- Do not treat shallow examples as practical depth
- Do not confuse lesson volume with instructional quality
